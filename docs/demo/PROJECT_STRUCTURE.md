# Структура проекта

## Основные папки

`app/` — маршруты Next.js App Router. Файлы `page.tsx` открывают страницы: главная панель, импорт, ОСВ, маппинг, корректировки, отчеты, аналитика, база знаний и журнал действий.

`components/` — React-компоненты интерфейса. В `components/pages/` лежит основная логика экранов, в `components/ui/` — переиспользуемые кнопки, карточки, бейджи и таблицы, в `components/layout/` — оболочка приложения с навигацией.

`lib/fixtures/` — синтетические демо-данные: компании, оборотно-сальдовые ведомости, IFRS-строки и начальный маппинг.

`lib/ifrs/` — упрощенные правила IFRS/MSFO. Каждый файл отвечает за отдельный стандарт или группу расчетов: IFRS 9, IFRS 15, IFRS 16, IAS 12, IAS 16, IAS 21, IAS 36.

`lib/statements/` — генераторы отчетности. Здесь формируются SOFP, P&L, упрощенный Cash Flow и влияние корректировок на строки отчетности.

`lib/export/` — экспорт отчетности в Excel, JSON и XML/XBRL-like.

`lib/store/` — локальное состояние приложения на localStorage: выбранная компания, ОСВ, маппинг, корректировки, роль и журнал действий.

`lib/mapping/` — логика сопоставления локальных счетов со строками IFRS-отчетности.

`lib/analytics/` — расчет коэффициентов и данных для графиков.

`docs/` — документация проекта. Папка `docs/demo/` содержит материалы для передачи студенту, записи видео и подготовки скриншотов.

`tests/` — Vitest-тесты для маппинга, IFRS-правил, отчетности, аналитики и экспорта.

`data/` — пример CSV-файлов для проверки импорта.

## Что открыть, если руководитель спросит “где код”

- Страницы интерфейса: `components/pages/`
- Демо-данные: `lib/fixtures/trial-balances.ts`
- Правила МСФО: `lib/ifrs/`
- Формирование отчетов: `lib/statements/`
- Экспорт: `lib/export/`
- Состояние приложения: `lib/store/app-store.tsx`
- Тесты: `tests/`

## Что важно объяснить

Проект intentionally локальный: данные хранятся в браузере, backend и база данных не используются. Это подходит для учебной демонстрации, потому что цель — показать workflow цифровой платформы и логику трансформации, а не создавать production-систему учета.
