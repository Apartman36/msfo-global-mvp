# MSFO.Global: памятка для демонстрации

## Что это за проект

MSFO.Global — учебный прототип локальной цифровой платформы для демонстрационной трансформации бухгалтерских данных в упрощенную IFRS/MSFO-отчетность. Проект подготовлен для темы: “Digital platform MSFO.Global: an integrated solution for implementing International Financial Reporting Standards (IFRS/MSFO) in Russia and Zambia.”

Репозиторий: https://github.com/Apartman36/msfo-global-mvp

Важная формулировка: это учебный прототип, а не промышленная бухгалтерская система. В приложении используются синтетические демо-данные и упрощенные правила МСФО.

## Что показывать руководителю

- Общий workflow: загрузка данных → ОСВ → маппинг → IFRS/MSFO-корректировки → отчетность → аналитика → экспорт.
- Выбор юрисдикции и компании: Россия/RUB и Zambia/ZMW.
- Положительный, но сниженный результат после корректировок: IFRS-прибыль меньше локальной прибыли, потому что применяются учебные корректировки.
- Отчеты SOFP, P&L и упрощенный Cash Flow.
- Экспорт Excel, JSON и XML/XBRL-like.
- Журнал действий как имитацию audit trail.

## Что не утверждать

- Не говорить, что это готовая промышленная IFRS-система.
- Не говорить, что расчеты подходят для реального аудита.
- Не говорить, что XML является валидным XBRL.
- Не говорить, что есть настоящая база данных, backend, ERP-интеграция или полноценная авторизация.
- Не утверждать, что демо-данные являются реальными данными компаний.

## Как запустить локально

```bash
git clone https://github.com/Apartman36/msfo-global-mvp.git
cd msfo-global-mvp
npm install
npm run dev
```

Открыть в браузере: http://localhost:3000

Если проект уже находится на компьютере:

```powershell
cd C:\Users\hustlePC\PycharmProjects\msfo-global-mvp
npm run dev
```

## Какие страницы есть

- Панель — KPI, компания, юрисдикция, прогресс workflow и последние действия.
- Импорт — загрузка XLSX/CSV и кнопка загрузки демо-данных.
- ОСВ — исходная оборотно-сальдовая ведомость.
- Маппинг — сопоставление локальных счетов со строками IFRS-отчетности.
- Корректировки — применение упрощенных правил IFRS 9, IFRS 15, IFRS 16, IAS 12, IAS 16, IAS 21, IAS 36.
- Отчеты — SOFP, P&L, Cash Flow и экспорт Excel/JSON/XML.
- Аналитика — коэффициенты, структура активов и графики корректировок.
- База знаний — краткие карточки стандартов.
- Журнал действий — события в локальном audit log.

## Как сбросить демо-данные

Нажать кнопку “Сбросить демо-данные” в верхней панели или “Загрузить демо-данные” на главной/странице импорта. Это вернет синтетический набор данных для выбранной компании.

## Как применить IFRS/MSFO-корректировки

1. Открыть страницу “Корректировки”.
2. Нажать “Применить правила”.
3. Проверить, что появились карточки/строки стандартов IFRS/IAS.
4. Вернуться на “Панель” или открыть “Отчеты”.

## Как экспортировать отчеты

1. Открыть страницу “Отчеты”.
2. При необходимости выбрать вкладку SOFP, P&L или Cash Flow.
3. Нажать Excel, JSON или XML/XBRL-like.
4. Файл будет скачан браузером.

## Пример файлов для импорта

В папке `data/` есть CSV-файлы:

- `data/demo_ru_trial_balance.csv` — пример ОСВ для России.
- `data/demo_zm_trial_balance.csv` — пример ОСВ для Замбии.

Их можно выбрать на странице “Импорт”, чтобы показать загрузку CSV.

## Какие файлы и папки важны

- `app/` — страницы приложения.
- `components/` — интерфейс и страницы.
- `lib/fixtures/` — синтетические демо-данные.
- `lib/ifrs/` — упрощенные правила МСФО.
- `lib/statements/` — генерация отчетов.
- `lib/export/` — Excel/JSON/XML экспорт.
- `lib/store/` — localStorage-состояние.
- `tests/` — автоматические проверки.
- `docs/demo/` — материалы для защиты, видео и скриншотов.

## Известные ограничения

Проект является учебным прототипом. Он показывает идею интегрированной платформы и последовательность трансформации, но не заменяет профессиональное бухгалтерское, налоговое или аудиторское суждение. Правила МСФО намеренно упрощены, данные синтетические, XBRL только имитируется, backend и база данных не используются.
