# Краткий рассказ для руководителя

Тема моей работы — цифровая платформа MSFO.Global как интегрированное решение для внедрения Международных стандартов финансовой отчетности в России и Замбии.

Идея проекта в том, чтобы показать, как компания может пройти понятный цифровой workflow трансформации: загрузить локальные бухгалтерские данные, сопоставить счета с IFRS-строками, применить корректировки по стандартам и получить отчетность в более международном формате.

МСФО важны, потому что они делают отчетность более сопоставимой для инвесторов, банков, партнеров и международных групп компаний. Для России это актуально из-за различий между локальным учетом и международной отчетностью. Замбия добавлена как вторая юрисдикция, чтобы показать, что платформа может поддерживать не одну страну, а разные валюты, компании и учетные контексты.

В демо используются две учебные компании: российская компания в рублях и компания из Замбии в квачах. Данные синтетические, то есть они не взяты из реальной бухгалтерии. Это сделано специально, чтобы безопасно показать процесс трансформации.

Сначала пользователь видит панель с ключевыми показателями. Затем можно открыть страницу импорта, посмотреть оборотно-сальдовую ведомость, перейти к маппингу счетов и применить упрощенные правила МСФО. В прототипе есть демонстрационные корректировки по IFRS 9, IFRS 15, IFRS 16, IAS 12, IAS 16, IAS 21 и IAS 36.

После применения правил система формирует три вида отчетов: отчет о финансовом положении, отчет о прибылях и убытках и упрощенный отчет о движении денежных средств. Также есть аналитика с коэффициентами и экспорт в Excel, JSON и XML/XBRL-like формат.

Практическая значимость прототипа в том, что он показывает не просто отдельный расчет, а целую логику цифровой платформы: данные, трансформация, контроль, отчетность, аналитика и экспорт в одном интерфейсе.

Ограничения тоже важны: это учебный прототип, правила МСФО упрощены, нет реальной ERP-интеграции, production-базы данных, настоящей авторизации и полноценной XBRL-валидации. Для бакалаврской работы это допустимо, потому что цель — показать концепцию, архитектуру и пользовательский сценарий, а не заменить промышленную учетную систему.

В дальнейшем проект можно развивать: добавить backend, базу данных, реальные роли пользователей, интеграцию с 1C, Xero или QuickBooks, полноценную XBRL-валидацию, расширенный набор стандартов и облачное развертывание.
